# 灵枢智衡项目：请导师提供或确认的内容清单

老师您好，我负责项目系统的具体实现。为了先完成针灸领域的基础版本，需要您这边提供或确认以下内容。

## 一、近期需要提供的内容

### 1. 首批针灸资料

请提供一批用于系统初始建设和测试的资料，格式可以是 PDF 或 Markdown，包括：

- 针灸相关科研论文；
- 指南、共识或规范性资料；
- 其他您认为应该纳入知识库的针灸资料。

如方便，请同时说明哪些资料最重要、最适合作为首批验证内容。

### 2. 知识范围和关注重点

请确认基础版本在针灸领域中优先覆盖哪些内容，例如：

- 重点疾病或症状；
- 重点穴位、穴位组合或治疗方式；
- 需要提取的治疗参数，如频率、疗程、时长等；
- 需要重点关注的疗效指标、不良反应或禁忌信息。

### 3. 判断知识是否正确的基本标准

系统会从资料中提取知识关系和证据内容。请提供或确认：

- 术语名称是否有统一写法，例如穴位、疾病、疗效指标的标准名称；
- 哪些文献类型或来源可以认为更可靠；
- 哪些信息必须由专业人员确认后才能使用；
- 出现不同论文结论不一致时，希望如何处理。

### 4. 用于验证系统的问题

请提供一些您希望系统能够回答的问题，作为开发后的效果验证参考。例如：

- 某种疾病常见的针灸干预方案有哪些证据支持？
- 某个穴位或穴位组合主要用于哪些病症？
- 某类针灸治疗的参数和疗效结局在文献中如何描述？

问题不需要很多，先提供一批具有代表性的即可。

### 5. 审核人员安排

系统会保留知识审核功能。后续请确定：

- 谁负责查看和确认系统抽取出的知识；
- 哪些内容需要重点审核；
- 系统整理出的结果以什么形式交给审核人员最方便。

当前开发可以先进行，不需要立即确定具体审核时间。

## 二、外部资料接口相关内容

如果后续需要系统定期从某个接口或渠道获取新资料，请在接口明确后提供：

- 接口地址或资料获取方式；
- 是否需要账号、密钥或授权；
- 请求时需要传入哪些条件；
- 返回的是 JSON 信息、文件，还是文件下载地址；
- 获取新资料的频率要求，例如每天、每周或手动触发；
- 可用于测试的一份真实返回样例。

在这些内容确定前，我会先实现人工上传资料的流程，以及预留可接入外部来源的位置。

## 三、后续扩展时再确认的内容

以下内容不影响当前针灸知识平台基础版本开发，可在后续需要时再提供或确认：

- 是否加入耳穴、经皮穴位电刺激、迷走神经刺激等其他领域；
- 是否与现有科研管理系统对接，以及需要交换哪些数据；
- 是否需要进一步支持科研方案生成、量表推荐等功能；
- 是否需要接入治疗设备或生理信号数据；
- 项目如果从内部科研使用转向对外展示或商用，需要新增哪些管理和合规要求。

## 四、我这边负责的工作

我将负责完成：

- 资料上传、解析和知识提取流程；
- 知识图谱构建、检索与网页对话功能；
- 新资料增量导入和管理后台；
- 模型调用、配置管理和后续模块接入的系统实现；
- 根据您提供的资料和标准，对功能进行迭代优化。

当前最希望优先收到的是：**首批针灸资料、基础版本重点关注范围、若干代表性验证问题**。这些内容到位后，我即可围绕真实资料推进实现和测试。
