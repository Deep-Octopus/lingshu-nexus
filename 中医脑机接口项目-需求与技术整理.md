# 灵枢智衡 (LingShu Nexus) — 需求与技术整理

> **项目名称**：灵枢智衡 | LingShu Nexus | 文件夹 `lingshu-nexus`
> **命名释义**："灵枢"取中医经典《灵枢》之名，专论针灸、经络、穴位，是项目核心知识域的理论源头；"智衡"取"AI 智能权衡辨证、平衡施治"之意，体现专家系统从多源知识中输出治疗方案的闭环逻辑；Nexus 意为"联结/枢纽"，呼应多知识域交叉关联、多 Agent 协作调度、评估端与干预端的闭环联通。

## 一、项目定位

构建一个**非侵入式闭环中医脑机接口**系统，核心逻辑为：

```
个体化评估（问诊 + 生理信号） → 知识图谱专家系统（大模型驱动） → 干预方案输出 → 刺激设备执行
```

最终目标是形成一个既能**辅助临床治疗决策**、又能**支撑科研实验设计**的智能体平台。项目对外以"中医脑机接口"的名义立项，作为张洁的开题方向（预计 2026 年 10 月前后）。

## 二、核心业务场景

### 2.1 临床治疗辅助
- 医生输入患者症状/病症信息
- 系统通过问诊分析，判断病症类型
- 调用知识图谱专家系统，输出治疗处方：
  - **针灸**：穴位组合方案
  - **经皮穴位电刺激**：穴位 + 电极贴敷位置
  - **迷走神经刺激**：刺激参数（频率、强度等）
  - **耳穴刺激**：基于耳穴理论的刺激位置（可扩展到迷走神经以外的耳部位置）
- 可选：通过摄像头辅助指导电极贴敷/穴位定位

### 2.2 科研实验设计辅助
- 输入研究方向，系统基于知识图谱生成：
  - 推荐使用的评估量表
  - 纳入/排除标准建议
  - 刺激参数选择建议
- 生成的方案可填充到现有的科研管理系统（小梅正在开发的 2.0 版本）中

### 2.3 科研文献自动获取与图谱构建
- 科研板块后续会接入外部接口，由对方提供指定主题、疾病、干预方式对应的 PDF、文献元数据或其他结构化材料
- 系统需要支持把外部文献接口配置成数据源，并按规则自动拉取：
  - 手动触发：管理员选择主题/时间范围后立即拉取
  - 定时触发：按天/周/月自动同步新文献
  - 增量触发：根据 DOI、PMID、文献哈希或更新时间判断是否已入库
- 拉取后的文献进入统一处理流水线：
  - 文件入库与元数据登记
  - PDF/HTML/Word/纯文本解析
  - 结构化知识抽取
  - 人工审核
  - 图谱增量更新
  - GraphRAG 索引刷新
- 该能力使知识图谱从一次性建设变成“持续更新的活证据图谱”，避免科研证据过时。

### 2.4 Agent Skill 能力管理
- 本项目中的 **skill** 指 Agent Skill，不只是普通业务标签或设备配置文件。
- Agent Skill 是给智能体读取和执行的能力包，通常由一个目录组成，核心文件是 `SKILL.md`，里面描述：
  - 这个 skill 何时被使用
  - agent 应该遵循的步骤
  - 可调用的脚本、模板、参考资料或工具约束
  - 输入输出格式、安全边界和失败处理方式
- 设备控制可以作为一种 Agent Skill，但不等同于全部 skill。后续可沉淀多类 skill：
  - 文献抽取 Skill：指导 agent 如何从 PDF 中抽取 PICO、穴位、参数、疗效、证据等级
  - 图谱构建 Skill：指导 agent 如何进行实体标准化、关系入库、冲突处理
  - 设备控制 Skill：指导 agent 如何把治疗方案转成设备 JSON
  - 科研方案 Skill：指导 agent 如何生成量表、纳排标准、实验参数建议
  - 专家审核 Skill：指导 agent 如何整理待专家确认的问题清单
- 后台需要支持 Skill 的上传、版本管理、启停、审核、测试运行和执行日志查看。

### 2.5 后台管理面板
- 系统需要建设后台管理面板，服务于张洁、学生团队、管理员和专家审核人员。
- 后台核心功能：
  - 用户与角色管理：管理员、研究者、学生、专家审核员、只读访客
  - 文献数据源管理：外部接口地址、认证方式、同步频率、同步范围
  - 定时任务管理：新增/暂停/恢复/立即执行/重试/查看日志
  - 文献入库管理：查看拉取状态、解析状态、抽取状态、审核状态
  - 知识图谱审核：实体合并、关系确认、冲突处理、证据等级修正
  - Agent Skill 管理：上传、版本、审核、启停、试运行、回滚
  - 设备与执行日志：设备控制 Skill、命令记录、回调结果、异常追踪
  - 系统监控：任务队列、模型调用量、Token 成本、失败率、数据库状态

## 三、技术架构

### 3.1 整体架构

```
┌──────────────────────────────────────────────┐
│                 用户交互层                     │
│  问诊界面 / 方案展示 / 科研工作台 / 后台管理    │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│              大模型智能体层                    │
│  多智能体协作 / Agent Skill 加载 / 任务路由    │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│           知识图谱专家系统（核心）              │
│  GraphRAG / 知识图谱查询 / 推理 / 方案推荐     │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│             科研数据与任务调度层                │
│  外部接口拉取 / 文献解析 / 定时任务 / 审核流     │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│              知识图谱数据层                    │
│  文献知识图谱（穴位-病症-参数-疗效关系网络）    │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│              干预执行层                        │
│  刺激设备（终极盒）/ 针灸 / 经皮电 / 迷走刺激   │
└──────────────────────────────────────────────┘
```

### 3.2 关键技术点

| 技术点 | 说明 |
|--------|------|
| **GraphRAG** | 基于知识图谱的检索增强生成，是系统的核心技术路线。需调研开源方案及具体落地路径 |
| **知识图谱构建** | 从高水平文献中抽取结构化知识（实体-关系-属性），构建穴位、病症、参数、疗效之间的关系网络 |
| **多智能体协作** | 多个子领域的智能体并行工作，最终汇总为一个完整的中医脑接口智能体 |
| **Agent Skill** | 将稳定可复用的任务流程封装成 `SKILL.md` 能力包，让 agent 按规范执行文献抽取、图谱构建、设备控制、科研方案生成等任务 |
| **定时任务管理** | 支持外部科研接口的周期拉取、文献解析、图谱增量构建、失败重试和执行日志 |
| **后台管理面板** | 管理用户、数据源、任务、文献、图谱审核、Agent Skill、设备与系统监控 |
| **专家系统** | 基于知识图谱的精准问答 + 治疗方案推理输出 |
| **闭环系统** | 评估 → 决策 → 干预 → 反馈，形成完整闭环 |

## 四、知识图谱覆盖的知识域

需要并行推进，每个方向由不同学生负责文献收集与梳理：

| 序号 | 知识域 | 文献规模（估） | 核心内容 |
|------|--------|---------------|----------|
| 1 | **迷走神经刺激（VNS）** | ~500 篇 | 刺激参数、频率机制、适应症、疗效 |
| 2 | **耳穴** | 待收集 | 耳穴定位、耳穴-病症对应关系、耳穴刺激理论 |
| 3 | **经皮穴位电刺激（TEAS）** | 待收集 | 穴位-电极贴敷、参数、适应症 |
| 4 | **传统针灸** | 待收集 | 穴位组合、手法、适应症（从高水平文献抽取） |
| 5 | **中药**（远期/陆教授合作） | — | 中药-经络-适应症关系网络 |

> 这些知识域之间存在交叉关系：迷走神经 ↔ 耳穴 ↔ 经皮穴位电 ↔ 传统针灸穴位，知识图谱需要把它们关联打通。

## 五、与现有工作的关系

| 维度 | 当前项目（孕妇健康管理） | 新项目（中医脑机接口） |
|------|------------------------|----------------------|
| 定位 | 健康管理（非临床治疗） | 临床治疗辅助 + 科研支撑 |
| 干预 | 无干预，仅评估 | 有干预（电刺激/针灸方案输出） |
| 评估方式 | 个体化问诊为主 | 问诊 + 生理信号（心电/脑电，后续加入） |
| 知识支撑 | 非知识图谱驱动 | 知识图谱专家系统驱动 |
| 战略优先级 | 可稍放缓 | **当前战略性重点** |

**汇合点**：两个项目最终合并为"中医脑机接口"——个体化评估 + 专家系统共性方案 → 个性化干预推荐。

## 九、外部参考

- OpenAI Skills 仓库对 Agent Skill 的定义：Agent Skills 是由说明、脚本和资源组成的文件夹，供 AI agent 发现并用于完成特定任务。参考：[openai/skills](https://github.com/openai/skills)
- GitHub Copilot 文档也采用类似模式：每个 skill 独立目录，包含 `SKILL.md`，可附带补充 Markdown 或脚本；当 Copilot 判断需要使用某个 skill 时，会把 `SKILL.md` 注入 agent 上下文。参考：[Adding agent skills for GitHub Copilot](https://docs.github.com/en/copilot/how-tos/copilot-on-github/customize-copilot/customize-cloud-agent/add-skills)
